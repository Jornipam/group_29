1) id : идентификатор, уникальное значение, позволяющее отличить один отчет о дефекте от другого
2) summary : название/краткое описание/ кратко отчечает на вопросы "Что произошло?" "Где это произошло?" "При каких условиях это произошло?"
3) description : развернутое описание, обязательно! описание фактического и ожидаемого результатов и ссылку на требование (если это возможно)
4) steps to reproduce, STR : описывают действия, которые необходимо выполнить для воспроизведения дефекта
5) reproducibility : воспроизводимость, always / sometimes
6) severity : важность ,степень ущерба, наносимого проекту (critical,major,medium,minor)
7) priority : срочность, как быстро дефект должен быть устранен ASAP-as soon as possible, high, normal, low
8) workaround : возможность обойти
9) comments, additional info : может содержать любые полезные для понимания и исправения данные
10) attachments : прикрепенные к отчету приложения (скрины,видео)
